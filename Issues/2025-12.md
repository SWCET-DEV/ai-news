## [Context Engineering 2.0 (11/06)](https://cobusgreyling.medium.com/context-engineering-2-0-456a1d3c1df1)
 > 요즘 AI 환경에서 ‘Context Engineering’이 점점 더 핵심 스킬이 되고 있잖아요. 그런데 우리 각자는 프로젝트 컨텍스트를 어떻게 구조화하고 관리하고 있을까요?
예를 들어, 컨텍스트 윈도우 안에서 정보를 어디에 두면 모델이 가장 잘 이해할지, 또는 ‘Lost in the Middle’ 문제를 어떻게 피하고 있는지 궁금해요. 재밌는 건, 상하이 연구 그룹에서 AI가 장기 컨텍스트를 스스로 정리하고, 필요하면 기억도 지우는 ‘Semantic Operating System’이라는 개념을 제안했다는 점이에요. 기사와 논문을 보면서—앞으로 우리가 컨텍스트를 어떻게 다뤄야 할지, 새로운 힌트를 한번 찾아보면 좋을 것 같습니다!
 > ``Context Engineering``, ``Semantic Operating System``

## [앤트로픽, 도구 호출 대신 코드 실행하는 'MCP' 출시 (11/10)](https://www.aitimes.com/news/articleView.html?idxno=203853)
 > Anthropic이 최근 흥미로운 변화를 시도했어요. 기존의 툴 호출 방식 대신, MCP(Model Context Protocol)를 활용한 코드 실행 기반 패턴으로 방향을 튼 건데요. 이 방식의 장점은 꽤 명확합니다. 에이전트가 여러 도구를 한꺼번에 참조할 때 생기던 컨텍스트 창 과부하, 토큰 비용 폭증, 응답 지연 같은 골칫거리들을 크게 줄일 수 있어요. 필요한 도구만 골라 로드하고, 대용량 데이터는 코드 쪽에서 미리 가공해 모델에 전달하니 비용도 아끼고 민감 정보 노출도 예방할 수 있죠. 실제 사례에선 기존에 약 15만 토큰을 쓰던 워크플로우가 2천 토큰 수준으로 떨어졌다고 하니… 거의 98.7% 할인 쿠폰 수준의 혁신입니다. 앞으로 에이전트 설계 패턴도 슬슬 이쪽으로 이동하는 흐름이 생기지 않을까 싶어요!
 > `` Code execution with MCP ``, ``Anthropic``
